Security Architecture

Hardened from the ground up

Every layer of HotelPro Nexus is secured by multiple defence mechanisms — from token expiry to IP-level audit trails.

OTP Verification

6-digit time-limited OTP sent via email. Auto-invalidated after use. Session is not created until OTP is verified.

Role-Based Access Control

3 isolated roles — Super Admin, Hotel Admin, Customer — each with separate sessions, routes, and decorators.

Password Reset Audit Trail

Every reset event (REQUEST, SUCCESS, FAILED, BLOCKED) is immutably logged with IP address and timestamp.

Token Expiry & Rate Limiting

Signed reset tokens expire automatically. Brute-force attempts are flagged as BLOCKED in the audit log.

Independent Session Middleware

Custom middleware ensures sessions are namespaced per role — an Admin session cannot bleed into a Customer session.

Email-First Authentication

Email is the unique login field instead of username. No guessable user IDs. Secure Django AbstractUser override.
